## 三、详细需求说明

### 3.1 P1-003 模拟面试模式

**需求背景**:
用户在实际面试前需要模拟演练，当前系统缺少此功能。

**功能描述**:
1. **随机抽题**: 从题库中随机抽取指定数量的题目
2. **计时答题**: 设置答题时间限制，模拟真实面试压力
3. **结果报告**: 答题结束后生成详细报告，包括正确率、用时分析、薄弱环节
4. **难度选择**: 可选择面试难度（初级/中级/高级）
5. **分类选择**: 可选择面试方向（前端/全栈/性能优化等）

**交互流程**:
```
选择模式 → 设置参数 → 开始答题 → 提交答案 → 查看报告
```

**验收标准**:
- [ ] 支持随机抽取 5/10/15/20 道题目
- [ ] 支持设置答题时间限制
- [ ] 答题过程显示倒计时
- [ ] 生成包含正确率、用时、建议的报告
- [ ] 报告可保存到学习记录

### 3.2 P1-004 AI 评分功能（暂不实现）

> **注意**: 此功能暂不纳入当前开发计划，后续根据需求再评估实现。

**需求背景**:
当前理论题评分基于关键词匹配，准确性有限，无法给出精准的改进建议。

**功能描述**:
1. **智能评分**: 使用大语言模型对用户答案进行评分
2. **详细反馈**: 指出答案的优点和不足
3. **改进建议**: 提供具体的改进方向和参考答案对比
4. **代码审查**: 对编程题代码进行质量评估

**技术方案**:
- 集成 OpenAI API 或本地部署的开源模型
- 设计评分 Prompt 模板
- 添加 API Key 配置（用户自行配置或服务端代理）

**验收标准**（暂不实现）:
- [ ] 理论题评分准确率提升至 80% 以上
- [ ] 提供详细的答案分析
- [ ] 给出具体的改进建议
- [ ] 支持代码质量评估

### 3.3 P1-005 学习计划制定

**需求背景**:
用户缺乏学习目标引导，容易半途而废。

**功能描述**:
1. **每日目标**: 设置每日学习题目数量
2. **学习计划**: 制定周计划、月计划
3. **进度追踪**: 显示计划完成进度
4. **提醒功能**: 未完成目标时提醒
5. **成就系统**: 完成目标获得成就徽章

**验收标准**:
- [ ] 支持设置每日学习目标
- [ ] 显示计划完成进度条
- [ ] 支持周计划、月计划
- [ ] 完成目标显示成就动画

### 3.4 P2-001 复习提醒功能

**需求背景**:
根据艾宾浩斯遗忘曲线，用户需要定期复习才能巩固记忆。

**功能描述**:
1. **复习算法**: 基于艾宾浩斯遗忘曲线计算复习时间
2. **复习队列**: 显示需要复习的题目列表
3. **复习提醒**: 到达复习时间时提醒用户
4. **复习记录**: 记录复习次数和效果

**复习间隔参考**:
- 第1次复习：学习后 1 天
- 第2次复习：学习后 3 天
- 第3次复习：学习后 7 天
- 第4次复习：学习后 14 天
- 第5次复习：学习后 30 天

**验收标准**:
- [ ] 自动计算复习时间
- [ ] 显示复习队列
- [ ] 支持手动标记已复习

### 3.5 P2-002 笔记系统

**需求背景**:
用户希望记录个人学习心得和笔记。

**功能描述**:
1. **题目笔记**: 每道题目可添加个人笔记
2. **Markdown 支持**: 支持 Markdown 格式
3. **笔记搜索**: 可搜索笔记内容
4. **笔记导出**: 可导出所有笔记

**验收标准**:
- [ ] 每道题目可添加/编辑/删除笔记
- [ ] 支持 Markdown 预览
- [ ] 笔记数据持久化存储
